# Audit SEO technique — Astro Cours

**Date :** 9 juillet 2026
**Périmètre :** code source (Next.js 16 App Router, trilingue fr/en/es), domaine `https://www.astro-cours.com`
**Méthode :** revue du code (métadonnées, i18n, sitemap, données structurées, maillage, perf, a11y). Chaque constat est référencé au fichier.

---

## 1. Synthèse

L'implémentation SEO est **globalement solide et bien pensée** : hreflang complet, canonicals absolus localisés, sitemap dynamique avec `lastmod` et alternates, données structurées sur tous les gabarits majeurs, `next/image` partout, polices non bloquantes, en-têtes de sécurité complets, un seul `<h1>` par page. Les gros écueils classiques sont déjà couverts.

Les axes d'amélioration restants sont **incrémentaux**, pas structurels. Les deux correctifs les plus urgents (redirections 308 + longueurs de titres) sont **déjà codés** et n'attendent qu'un déploiement.

### Ce qui est déjà bien fait (à ne pas casser)

- **hreflang** fr-FR / en-US / es-ES + `x-default` sur tous les types de page (`lib/seo.ts`, `lib/compatibility.ts`), et en-têtes `Link` middleware volontairement désactivés pour éviter les hreflang non canoniques (`i18n/routing.ts:25`).
- **Canonicals** absolus et localisés via `buildMeta` / `pillarUrl` / `localizedPathUrl`.
- **Sitemap** (`app/sitemap.ts`) : signes, planètes, maisons, paires de compatibilité, articles de blog, tags, pages annexes — avec `lastmod` et alternates hreflang par URL.
- **Données structurées** présentes sur les gabarits piliers (`Course`/`Article` + `BreadcrumbList` + `FAQPage`), compatibilité (3 blocs JSON-LD), blog (`Article`, `CollectionPage`), accueil (`@graph` Organization + Person).
- **Performance** : `next/image` partout (aucun `<img>` brut), préchargement LCP optimisé, polices `display:"optional"` non bloquantes, formats AVIF/WebP, `Vary: Accept-Encoding`.
- **Un seul `<h1>`** par page sur tous les gabarits.

## 3. Détail des constats

### P0 — Déployer les correctifs déjà codés
Les redirections 308 (`next.config.mjs` → `compatibilityRedirects()`) et les titres raccourcis (≤ 60 car.) sont en place dans le code mais **pas encore en production**. Tant que le déploiement n'est pas fait, Ahrefs continuera de remonter « Meta refresh redirect », « H1 vide » et « Title too long » sur les 27 URLs concernées. **Action : redéployer sur Vercel, puis relancer un crawl Ahrefs.**

### P1 — 7 pages annexes absentes du footer
Le footer (`components/layout/footer/Footer.tsx`) liste 13 des 20 pages annexes. **Manquent :** `astro-psychologie`, `astrologie-horaire`, `astrologie-medicale`, `astrologie-mondiale`, `maisons-dominantes`, `signes-dominants`, `significateurs`. Elles restent accessibles via la NavBar et le sitemap, donc non orphelines, mais reçoivent moins de jus interne et une priorité de crawl plus faible. **Action : ajouter ces 7 entrées au tableau des annexes du footer** pour uniformiser la couverture avec la NavBar.

### P1 — Fil d'Ariane visible manquant sur les pages annexes
Une seule page annexe (`cuspides-des-maisons`) rend le composant `<Breadcrumbs>` ; les 19 autres n'ont qu'un fil d'Ariane en JSON-LD, sans version visible à l'écran. Les pages piliers, elles, affichent le fil d'Ariane. **Action : ajouter `<Breadcrumbs>` (Accueil → hub → page) sur les 19 pages annexes.** Bénéfices : contexte de navigation, ancres internes supplémentaires, cohérence UX, repère ARIA.

### P2 — Image Open Graph unique pour ~25 gabarits
`/og/cover.jpg` est référencée 25 fois : accueil, tous les hubs, les 20 pages annexes, les pages `signes/[slug]` et la compatibilité utilisent toutes la même image de partage. Seuls le blog (couverture de l'article), les planètes (`planetHeroSrc`), les maisons (`/images/maisons/...`) et le dictionnaire ont une image dédiée. Partages sociaux et vignettes Discover peu différenciés. **Action recommandée : générer des images OG dynamiques** via une route `opengraph-image.tsx` (titre + thème de la page) pour les gabarits `signes/[slug]`, compatibilité et annexes — solution la plus rentable, sans créer d'images à la main.

### P2 — Titre `points-fictifs` trop long + double tiret
`app/[locale]/(annexe)/points-fictifs/content.tsx:70` → « Points fictifs — Vertex, Fortune, Chiron : méthode » (50 car.) reçoit le suffixe marque, soit **64 car.** (> 60), avec en plus deux tirets cadratin (celui du titre + celui de la marque). C'est la seule page annexe au-dessus de 60. **Action : soit passer par `title: { absolute }` (comme `lilith`), soit raccourcir en « Points fictifs : Vertex, Fortune, Chiron ».** (À noter : `aspects` est pile à 60, à surveiller.)

### P2 — E-E-A-T du blog
Le JSON-LD `Article` du blog (`components/blog/ArticleLayout.tsx`) référence `AUTHOR_PERSON` mais sans relier l'auteur à l'éditeur. **Action : ajouter `publisher: PUBLISHER_ORG` et/ou `author.worksFor: PUBLISHER_ORG`** pour renforcer le signal d'autorité (l'accueil le fait déjà via son `@graph`).

### P3 — Flux RSS/Atom
Aucun `feed.xml` / `rss.xml`. **Action : ajouter une route `app/feed.xml/route.ts`** listant les articles, et un `<link rel="alternate" type="application/rss+xml">` dans le `<head>`. Impact SEO indirect (découvrabilité, agrégateurs).

### P3 — Lien d'évitement (skip-link)
`app/[locale]/layout.tsx` n'expose pas de lien « Aller au contenu », alors que chaque page a bien `id="main-content"`. **Action : ajouter `<a href="#main-content" class="sr-only focus:not-sr-only">…</a>`** juste après l'ouverture du body. Gain d'accessibilité (facteur indirect).

### P3 — `trailingSlash` implicite
Non défini dans `next.config.mjs` (défaut `false`, correct). **Action : le fixer explicitement `trailingSlash: false`** pour verrouiller l'intention et éviter toute ambiguïté `/blog` vs `/blog/`.

### P3 — Maillage contextuel piliers → annexes
Le maillage hub → enfants est bon. Opportunité long terme : ajouter des liens **contextuels** depuis les pages piliers vers les annexes pertinentes (ex. page d'un signe → `/les-decans`, `/signes-dominants` ; page d'une planète → `/retrogrades`, `/maitrises`). Cela répartit mieux le jus vers les annexes et renforce la thématisation.

## 6. Constats écartés après vérification

Ces points, parfois suspectés, ont été **vérifiés et sont conformes** — aucune action nécessaire :

- Les pages `signes/[slug]`, `planetes/[slug]`, `maisons/[slug]` **émettent bien** du JSON-LD (1 bloc chacune).
- Les titres des pages `signes/[slug]` utilisent `title: { absolute }` : ils **ne dépassent pas 60 car.** (pas de suffixe marque ajouté).
- Les pages de tags du blog émettent bien un `CollectionPage`.
- `robots.ts` et le sitemap couvrent l'ensemble des pages.
